# Como Configurar o Sentry com o Leaf

O [Sentry](https://sentry.io/) é um serviço poderoso para rastreamento de erros em tempo real e monitoramento de desempenho. Integrá-lo ao seu servidor Leaf permite capturar erros e exceções automaticamente, facilitando muito o diagnóstico e a correção de problemas que ocorrem no servidor.

Este guia explica como configurar o Leaf para enviar relatórios de erros ao seu projeto no Sentry.

## Pré-requisitos

1.  **Conta no Sentry:** Você precisa de uma conta em [sentry.io](https://sentry.io/). Eles oferecem planos gratuitos adequados para muitos servidores.
2.  **Projeto no Sentry:** Crie um novo projeto dentro da sua organização no Sentry.
    - Quando solicitado a escolher a plataforma, selecione **Java**. Se Java não estiver visível imediatamente, você pode selecionar "Other" ou pesquisar por ele. A escolha específica da plataforma afeta principalmente as instruções de configuração inicial no lado do Sentry, mas o Leaf gerencia a integração por conta própria.
3.  **DSN (Data Source Name):** Após criar seu projeto, acesse suas configurações. Encontre a seção "Client Keys (DSN)". Copie a string DSN — ela se parece com uma URL (ex.: `https://xxxxxxxxxxxxxxxxxxxxxxxx@o######.ingest.sentry.io/#######`).

## Etapas de Configuração

1.  **Localize a Configuração do Leaf:** Abra o arquivo de configuração principal do seu servidor Leaf. Normalmente é o `leaf-global.yml`, localizado no diretório raiz do servidor ou em uma subpasta `config`.

2.  **Encontre a Seção do Sentry:** Dentro do arquivo de configuração, localize a seção `sentry:`. Por padrão, ela se parece com isto:

    ```yaml
    sentry:
        # Sentry DSN for improved error logging, leave blank to disable,
        # Obtain from https://sentry.io/
        dsn: ""
        # Logs with a level higher than or equal to this level will be recorded.
        log-level: WARN
        # Only log with a Throwable will be recorded after enabling this.
        only-log-thrown: true
    ```

3.  **Configure as Opções:**
    - **`dsn`:**
        - Substitua as aspas simples vazias (`''`) pelo DSN que você copiou das configurações do seu projeto no Sentry. Certifique-se de que está entre aspas simples.
        - **Exemplo:** `dsn: 'https://xxxxxxxxxxxxxxxxxxxxxxxx@o######.ingest.sentry.io/#######'`
        - _Se deixar em branco, a integração com o Sentry será desativada._

    - **`log-level`:**
        - Determina o nível mínimo de severidade das mensagens de log que o Leaf enviará ao Sentry. Os níveis comuns incluem `ERROR`, `WARN`, `INFO`, `DEBUG`.
        - O padrão `WARN` significa que apenas mensagens registradas como avisos ou erros (que normalmente incluem exceções) serão enviadas.
        - Você pode definir como `ERROR` se quiser apenas erros críticos. Definir um nível mais baixo (como `INFO`) pode enviar muito mais dados e pode não ser desejável, a menos que você esteja depurando ativamente.
        - **Recomendação:** Comece com `WARN` (o padrão).

    - **`only-log-thrown`:**
        - Se definido como `true` (o padrão), o Leaf _somente_ enviará mensagens de log ao Sentry se elas incluírem explicitamente um `Throwable` Java (como uma exceção ou erro com rastreamento de pilha). Isso é altamente recomendado, pois concentra o Sentry em erros reais do código.
        - Se definido como `false`, _qualquer_ mensagem de log que atenda ao limite de `log-level` será enviada, mesmo que seja apenas texto informativo sem erro. Isso pode gerar muito ruído no Sentry.
        - **Recomendação:** Mantenha como `true` (o padrão).

4.  **Salve e Reinicie:**
    - Salve as alterações no arquivo de configuração.
    - Reinicie completamente o seu servidor Leaf para que as novas configurações entrem em vigor.

## Solução de Problemas

- **Erros Não Aparecem:**
    - Verifique novamente se o `dsn` foi copiado corretamente e está entre aspas simples no arquivo YAML.
    - Certifique-se de que o `log-level` não está definido muito alto (ex.: se definido como `ERROR`, avisos não serão enviados).
    - Verifique se `only-log-thrown` está adequado para os erros que você espera ver.
    - Confirme que o servidor foi completamente reiniciado após alterar a configuração.
    - Verifique se o firewall do seu servidor pode estar bloqueando conexões de saída para `ingest.sentry.io` na porta 443 (HTTPS).
- **Muitos Problemas:**
    - Aumente o `log-level` (ex.: de `WARN` para `ERROR`).
    - Certifique-se de que `only-log-thrown` está definido como `true`.
