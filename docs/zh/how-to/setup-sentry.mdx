# 如何在 Leaf 中配置 Sentry

[Sentry](https://sentry.io/) 是一项强大的实时错误追踪和性能监控服务。将其与您的 Leaf 服务器集成后，可以自动捕获错误和异常，从而大大简化诊断和修复服务器问题的过程。

本指南说明如何配置 Leaf 以将错误报告发送至您的 Sentry 项目。

## 前提条件

1.  **Sentry 账户：** 在 [sentry.io](https://sentry.io/) 上创建一个账户。他们提供适合大多数服务器的免费套餐。
2.  **Sentry 项目：** 在您的 Sentry 组织中创建一个新项目。
    - 当被询问平台时，选择 **Java**。如果 Java 不立即可见，您可以选择"其他"或搜索。具体平台选择主要影响 Sentry 端的初始设置说明，但 Leaf 自行处理集成。
3.  **DSN（数据源名称）：** 项目创建后，进入其设置页面。找到"Client Keys (DSN)"部分，复制 DSN 字符串——它看起来像一个 URL（例如 `https://xxxxxxxxxxxxxxxxxxxxxxxx@o######.ingest.sentry.io/#######`）。

## 配置步骤

1.  **找到 Leaf 配置文件：** 打开您的 Leaf 服务器主配置文件，通常是位于服务器根目录或 `config` 子文件夹中的 `leaf-global.yml`。

2.  **找到 Sentry 配置节：** 在配置文件中找到 `sentry:` 部分，默认情况下如下所示：

    ```yaml
    sentry:
        # Sentry DSN for improved error logging, leave blank to disable,
        # Obtain from https://sentry.io/
        dsn: ""
        # Logs with a level higher than or equal to this level will be recorded.
        log-level: WARN
        # Only log with a Throwable will be recorded after enabling this.
        only-log-thrown: true
    ```

3.  **配置各项设置：**
    - **`dsn`：**
        - 将空的单引号（`''`）替换为从您的 Sentry 项目设置中复制的 DSN。确保用单引号括起来。
        - **示例：** `dsn: 'https://xxxxxxxxxxxxxxxxxxxxxxxx@o######.ingest.sentry.io/#######'`
        - _如果留空，Sentry 集成将被禁用。_

    - **`log-level`：**
        - 此项决定 Leaf 发送到 Sentry 的日志消息的最低严重级别。常见级别包括 `ERROR`、`WARN`、`INFO`、`DEBUG`。
        - 默认的 `WARN` 表示只有以警告或错误级别记录的消息（通常包含异常）才会被发送。
        - 如果只需要关键错误，可以设置为 `ERROR`。设置为更低级别（如 `INFO`）会发送更多数据，除非您正在主动调试，否则可能并不理想。
        - **建议：** 从默认的 `WARN` 开始。

    - **`only-log-thrown`：**
        - 设置为 `true`（默认值）时，Leaf 只会将明确包含 Java `Throwable`（例如带有堆栈跟踪的异常或错误）的日志消息发送到 Sentry。强烈建议保持此设置，因为它使 Sentry 专注于实际的代码错误。
        - 设置为 `false` 时，任何满足 `log-level` 阈值的日志消息都会被发送，即使只是没有错误的普通信息文本，这可能会在 Sentry 中产生大量噪音。
        - **建议：** 保持默认的 `true`。

4.  **保存并重启：**
    - 保存对配置文件的更改。
    - 完全重启您的 Leaf 服务器以使新设置生效。

## 故障排查

- **错误未出现：**
    - 仔细检查 `dsn` 是否已正确复制并在 YAML 文件中用单引号括起来。
    - 确保 `log-level` 设置不过高（例如，若设置为 `ERROR`，警告将不会被发送）。
    - 确认 `only-log-thrown` 的设置适合您期望看到的错误类型。
    - 确保更改配置后服务器已完全重启。
    - 检查服务器防火墙是否阻止了到 `ingest.sentry.io` 443 端口（HTTPS）的出站连接。
- **问题过多：**
    - 提高 `log-level`（例如从 `WARN` 改为 `ERROR`）。
    - 确保 `only-log-thrown` 设置为 `true`。
